## 1. 产品定位与目标用户

**一句话**：一个给电影版权采购者用的"全球电影情报雷达"——每天自动把全世界值得关注的新片、票房、行业动态、电影节片单汇到一处，手机随时看。

**目标用户**：电影版权买家（核心用户李超）。其工作特点决定了产品的核心原则：

- **信息及时性 > 信息完整性**：片单/新片一出现就要知道并跟进，等节展闭幕再汇总"黄花菜都凉了"。
- **要决策线索，不要做报告**：用户要的是"哪些新片值得我去了解/接洽"，不是统计图表。
- **中文优先**：用户看中文效率最高，外文资讯必须翻译。
- **零维护负担**：用户是技术小白，系统要自动运转，日常只需打开网页看。

**北极星指标**：用户能否在"片单/新片公布的当天"就看到并开始跟进。

## 2. 系统总览与角色

| 角色 | 做什么 | 在哪 |
|---|---|---|
| 抓取端 | 每天自动抓数据写云端 | Mac mini（家里，自动） |
| 数据库 | 存所有数据 | Supabase（云端） |
| 查看端 | 看数据、做筛选标记 | 网页书签（手机/电脑） |

用户日常只接触**查看端**（一个网址）。抓取端装好后无感运行。

## 3. 页面结构

单页应用，顶部一个标题栏 + 四板块导航，下方为当前板块内容。顶部显示"更新于 …"（最近一次成功抓取时间，异常时显示⚠️被限速）。

```
顶栏：阿尔戈斯 · 全球电影情报站            更新于 2026-06-13 09:12
导航：[🎬 豆瓣情报] [💰 全球票房] [📰 媒体资讯] [🏆 电影节]
─────────────────────────────────────────────
            （当前板块的筛选条 + 卡片列表）
```

## 4. 功能模块

### 模块一：🎬 豆瓣情报（主筛片工作台）
- **用途**：从豆瓣每日新片里筛出值得引进/关注的片子，并跟踪其口碑成长。
- **筛选条**：年份(2026/2027/全部) · 状态(待筛/⭐重点关注/保留/淘汰/全部，带实时计数) · 国家下拉(按片量排序) · 排序(豆瓣分/IMDb分/有分优先/今日新增5星/今日新增短评) · 关键词搜索(片名/原名/导演)。
- **卡片信息**：片名+原名、国家·年份·片长·类型、导演/主演、豆瓣分、IMDb 分、★5 人数及今日增量、短评数及今日增量。
- **核心交互（业务逻辑）**：状态机流转，点击即写回云端——
  - 待筛 → ⭐重点 / 保留 / 淘汰
  - ⭐重点关注 → 转保留 / 淘汰
  - 保留 → ⭐转重点 / 淘汰
  - 淘汰 → 恢复待筛
- **意义**：把"看过的判断"沉淀成一份持续维护的"重点/保留"清单。

### 模块二：💰 全球票房
- **用途**：看各国市场当周票房表现，发现有票房信号的新片（尤其本土类型片）。
- **数据**：Box Office Mojo 约 80+ 市场，每市场最新周末 Top3，按市场分组展示。
- **筛选条**：市场下拉 · 关键词搜索（片名/市场）。
- **卡片信息**：排名#、片名、市场·周末档期·上映第几周、周末票房、累计票房；**匹配到豆瓣后补**：中文名、国家·片长·类型、导演主演、豆瓣分、IMDb 分。
- **业务逻辑**：匹配到筛片库的片，卡片上可直接做保留/淘汰标记（与豆瓣板块同一状态体系）。

### 模块三：📰 媒体资讯
- **用途**：盯行业动态，尤其"某节展公布片单/某片入围"这类一手消息。
- **数据源**：Variety / Deadline / THR / IndieWire / TheWrap / ScreenDaily / Cineuropa / FilmNewEurope（外媒）+ 界面文娱 / 时光网（中文）。
- **业务逻辑**：
  - **中文优先**：外文标题/摘要自动机器翻译，中文为主、原文小字；中文源直接显示。
  - **🎯节展雷达过滤**：标题/译文命中片单/入围/获奖/各大节展名等关键词时打标，可一键只看这类消息。
- **筛选条**：来源分段（全部 / 🎯节展雷达 / 各媒体）· 关键词搜索。保留近 30 天。

### 模块四：🏆 电影节（情报雷达）
- **用途**：把电影节从"事后查"变成"提前盯"——知道每个节展**下一次片单什么时候公布**，到点第一时间跟进。
- **数据**：30 个全球重点电影节/奖项/市场，分 S/A/B 级。
- **业务逻辑**：
  - 页面顶部 **📡情报雷达**：列出"接下来要第一时间盯的片单节点"（lineup_announce 带★的，如威尼斯7月下旬、洛迦诺7月上旬、多伦多7月底等）。
  - 每个节展卡片：级别、中英文名、国家城市、常规档期、2026 届期、📅片单节点、已回填的入围/获奖片单（可展开）。
- **现状**：已闭幕的戛纳/柏林/奥斯卡等已回填完整片单与获奖；进行中/未来的待公布后回填。

## 5. 典型用户流程

**流程 A · 每日盯盘（主路径，每天 1–2 分钟）**
1. 手机打开书签 → 看顶部"更新于今天"确认数据新鲜。
2. 进豆瓣情报 → 状态选"待筛" → 按国家/评分聚焦一批 → 逐个点 保留/⭐重点/淘汰。
3. 进媒体资讯 → 点🎯节展雷达 → 扫一眼有没有新片单公布。

**流程 B · 片单公布日（高价值事件）**
1. 资讯雷达或电影节页发现某节展片单公布。
2. 通知 Claude（新对话）联网核实并回填该节展 festival_films。
3. 回到电影节板块查看片单 → 挑感兴趣的片去了解/接洽。

**流程 C · 票房发现**
1. 进全球票房 → 选目标市场 → 看 Top3 有无陌生本土片有票房信号。
2. 卡片直接看豆瓣/IMDb 分与类型 → 值得的点"保留"纳入清单。

**流程 D · 系统维护（低频）**
- 顶部出现⚠️被限速 → 重取豆瓣 cookie 更新到 mini（见保姆级步骤"cookie 失效怎么办"）。
- 想立刻抓一次 / 改抓取时间 → mini 终端操作（保姆级步骤末尾）。

## 6. 设计原则（约束）

- **及时优先**：所有功能围绕"尽早暴露新片/新片单"，前瞻雷达 > 事后汇总。
- **中文化**：凡外文必翻译，降低阅读成本。
- **决策导向**：卡片给的是"要不要跟进"所需的判断信息（分数、类型、主创、票房信号），不堆砌无关数据。
- **低维护**：抓取全自动；查看端无需登录、无需 App，一个网址；零运营成本。
- **数据解耦**：抓取与查看互不影响，单点故障不致命。

---

## 7. 暂不做 / 边界

- 不做票务、排片、版权交易撮合——只做"情报发现"。
- 不做数据可视化报表/BI——用户要线索不要图表。
- 不做多人权限体系（当前自用；如需团队协作再加）。
- 不做短片/剧集/综艺收录（按片长与类型规则过滤）。
- 不在机房 IP / 云函数抓豆瓣（会被封）。
