# Mac mini 保姆级部署步骤（李超专用）

> 你的情况：全新 Mac mini，用 git 拉代码。仓库地址、密钥都已替你填好。
> 全程约 15 分钟。**每一步做完看一眼"预期结果"那行，对得上再做下一步。**
> 任何一步屏幕和教程对不上，截图发我。

---

## 准备：先在「主力 Mac」上推送一次（1 分钟）

mini 要拉到最新代码，所以先在你常用的那台 Mac 上：
1. 打开 **GitHub Desktop**。
2. 左下角如果有改动文件列表，在 Summary 框随便写一句（如 `mini部署`）。
3. 点蓝色 **Commit to main** → 右上点 **Push origin**。
4. 看到没有"未推送"提示了就行。

> 如果 GitHub Desktop 里看不到任何改动、也没有 Push 按钮，说明之前已经推过了，直接跳到下面 mini 的步骤。

---

# 以下全部在「Mac mini」上操作

## 第 1 步：打开「终端」

- 按 `Command(⌘) + 空格`，弹出搜索框，输入 `终端` 两个字，回车。
- 会出现一个白底或黑底的窗口，能打字。**后面所有命令都贴进这个窗口，每贴一行按一次回车。**

> 贴命令方法：在本文里选中一行 → `⌘C` 复制 → 点终端窗口 → `⌘V` 粘贴 → 回车。

---

## 第 2 步：装 Node（电脑的"运行环境"）

1. 先看装没装过。贴这行回车：
   ```
   node -v
   ```
   - **预期结果**：显示 `v22.x.x` 或更大的数字 → 已经有了，**跳到第 3 步**。
   - 如果显示 `command not found` → 继续往下装。
2. 用 Safari 打开网址：**https://nodejs.org**
3. 页面中间有个大按钮，写着 **LTS**（推荐版），点它下载一个 `.pkg` 文件。
4. 下载完在「下载」里双击这个 `.pkg`，一路点「继续 / 同意 / 安装」，输一次开机密码，装完。
5. **把终端窗口关掉，重新打开一个**（第 1 步的方法），再贴一次 `node -v`。
   - **预期结果**：这次能看到 `v22.x.x` 之类的版本号。

---

## 第 3 步：把项目代码拉到 mini

逐行贴、逐行回车：
```
cd ~
```
```
git clone https://github.com/lichao1816-lab/douban-cloud.git
```
- **预期结果**：屏幕滚动几行 `Cloning... / Receiving objects... done`，最后回到能输入的状态。
- 如果提示要登录 GitHub：照提示输你的 GitHub 用户名和密码（密码那里可能要用 token，卡住就截图发我）。

然后进入项目文件夹：
```
cd ~/douban-cloud
```
- **预期结果**：命令行最前面会出现 `douban-cloud` 字样。

---

## 第 4 步：放入"钥匙文件" .env

钥匙文件（含密钥+豆瓣登录信息）在主力 Mac 上，文件名 `env-for-mini.txt`，在 `阿尔戈斯计划` 文件夹里。

**A. 先从主力 Mac 把它 AirDrop 给 mini：**
1. 在主力 Mac 的访达里找到 `阿尔戈斯计划` 文件夹 → 里面的 `env-for-mini.txt`。
2. 右键它 → 「**共享**」→「**隔空投送 AirDrop**」→ 选你的 Mac mini → 发送。
3. 在 mini 上点「接受」。文件会进 mini 的「下载」文件夹。

**B. 回到 mini 的终端，把它变成 .env：**
```
mv ~/Downloads/env-for-mini.txt ~/douban-cloud/.env
```
- **预期结果**：没有任何报错、直接回到能输入的状态，就是成功了（这条命令成功时是"安静"的）。
- 如果提示 `No such file`：可能 AirDrop 存到了别处，或文件名不一样。在 mini 终端贴 `ls ~/Downloads` 看下载夹里到底叫什么名字，截图发我。

> 没法 AirDrop？告诉我，我教你手动新建 .env。

---

## 第 5 步：装"每天自动抓"的定时任务

决定你想**每天几点抓**（24 小时制，挑一个 mini 一定开着的时间，比如上午 10 点）。
把下面的 `10:00` 换成你要的时间，贴进去回车：
```
bash setup-macmini.sh 10:00
```
- **预期结果**：屏幕打印一串 ✅，最后出现 **`部署完成 ✅`**，上面还会有一行 `sudo pmset repeat ...`（先别管，第 7 步用）。
- 如果出现 `❌ 未找到 .env` → 第 4 步没成功，回去检查。
- 如果出现 `❌ 没找到 node` → 第 2 步没装好，重装 Node。

---

## 第 6 步：立刻测试一次（不用等到点）

两行分别贴、回车：
```
launchctl start com.argos.douban.daily
```
```
tail -f logs/daily.out.log
```
- 屏幕开始滚动中文日志。**耐心等几分钟**（它在抓豆瓣 + 票房 + 资讯）。
- **预期结果**：最后出现 `今日汇总`，并且有一行 `是否被豆瓣限速: 否`。看到这两个就成功了！🎉
- 看完按 `Control(⌃) + C` 退出滚动（不会影响任务）。

> 如果出现 `是否被豆瓣限速: 是` → cookie 可能旧了，告诉我，我带你重取一段（5 分钟）。

---

## 第 7 步：让 mini 永不睡眠（很关键，否则到点不跑）

第 5 步结尾打印过一行 `sudo pmset repeat wakeorpoweron ...`（时间已按你设的自动算好）。
把**那一行**复制、贴进终端、回车，会要你输开机密码（输的时候屏幕不显示，正常，输完回车）：
```
（示例，以你屏幕上打印的那行为准）
sudo pmset repeat wakeorpoweron MTWRFSU 09:55:00
```
- **预期结果**：没报错就行。这样 mini 每天会自己提前 5 分钟醒来等着抓。

**再加一道保险**（图形界面）：
- 点屏幕左上角 苹果标志 → 「系统设置」→ 左侧找「电池」或「节能」→ 把「**防止电脑自动进入睡眠**」打开。

---

## 第 8 步：主力 Mac 停掉自己的抓取（避免两台打架）

回到**主力 Mac** 的终端，贴这行回车：
```
launchctl unload ~/Library/LaunchAgents/com.argos.douban.daily.plist
```
- 主力 Mac 之前没装过定时任务的话，会提示找不到文件，**忽略即可**。
- 目的：以后只让 mini 抓，两台同时抓豆瓣容易被封。

---

## 大功告成 ✅ 怎么确认长期正常？

- 以后每天到点，mini 自动抓，自动写云端，**完全不用你管、不花钱**。
- 想确认某天跑了没：打开你的手机/电脑书签 **https://lichao1816-lab.github.io/douban-cloud/** ，顶部「更新于 …」是不是今天、有没有「⚠️被限速」。
- 想手动补抓一次：mini 终端 `launchctl start com.argos.douban.daily`。

## 常见问题

**Q：cookie 多久会过期？过期了怎么办？**
A：几周到几个月。表现是网页顶部出现「⚠️被限速」。到时候告诉我，我带你 5 分钟重取一段、重装一下就好。

**Q：mini 重启了 / 停电了，任务还在吗？**
A：在。定时任务装一次就长期有效，重启后照常。

**Q：我改了主意想换抓取时间？**
A：mini 终端里重新跑一次 `bash setup-macmini.sh 新时间`，再按打印的新 pmset 行执行一次即可。
